# 歌手相关接口文档

## 功能概述
提供两个歌手相关接口：
1. **歌手搜索接口**：判断指定歌手是否存在于系统中
2. **歌手音乐查询接口**：根据歌手名称获取该歌手的所有音乐（与 `/files` 接口格式一致）

---

## 接口 1: 歌手搜索

### API 接口

**请求方式**: POST  
**路径**: `/artist/search`  
**Content-Type**: `application/json`

#### 请求参数
```json
{
  "artist": "歌手名称"
}
```

#### 响应格式
```json
{
  "exists": true/false
}
```

#### 示例

**示例 1: 搜索存在的歌手**
```bash
curl -X POST http://localhost:8080/artist/search \
  -H "Content-Type: application/json" \
  -d '{"artist":"周杰伦"}'
```
响应:
```json
{
  "exists": true
}
```

**示例 2: 搜索不存在的歌手**
```bash
curl -X POST http://localhost:8080/artist/search \
  -H "Content-Type: application/json" \
  -d '{"artist":"不存在的歌手"}'
```
响应:
```json
{
  "exists": false
}
```

**示例 3: 空参数验证**
```bash
curl -X POST http://localhost:8080/artist/search \
  -H "Content-Type: application/json" \
  -d '{"artist":""}'
```
响应:
```
Artist parameter is required
```

## 接口 2: 根据歌手查询音乐

### API 接口

**请求方式**: POST  
**路径**: `/music/artist`  
**Content-Type**: `application/json`

#### 请求参数
```json
{
  "artist": "歌手名称"
}
```

#### 响应格式
返回格式与 `/files` 接口完全一致，为 `FileListItem` 数组：

```json
[
  {
    "path": "音乐文件路径",
    "duration": "时长（秒）",
    "artist": "歌手名称（可选）",
    "cover_art_url": "封面图片URL（可选）"
  }
]
```

#### 示例

**示例 1: 查询周杰伦的音乐**
```bash
curl -X POST http://localhost:8080/music/artist \
  -H "Content-Type: application/json" \
  -d '{"artist":"周杰伦"}'
```
响应:
```json
[
  {
    "path": "千里之外/千里之外.mp3",
    "duration": "254.28 seconds",
    "artist": "周杰伦&费玉清",
    "cover_art_url": "http://slcdut.xyz:8080/uploads/千里之外/千里之外_cover.png"
  },
  {
    "path": "彩虹/彩虹.mp3",
    "duration": "261.96 seconds",
    "artist": "周杰伦",
    "cover_art_url": "http://slcdut.xyz:8080/uploads/彩虹/彩虹_cover.png"
  }
  // ... 更多歌曲
]
```

**示例 2: 查询林俊杰的音乐**
```bash
curl -X POST http://localhost:8080/music/artist \
  -H "Content-Type: application/json" \
  -d '{"artist":"林俊杰"}'
```
返回 79 首歌曲

**示例 3: 查询不存在的歌手**
```bash
curl -X POST http://localhost:8080/music/artist \
  -H "Content-Type: application/json" \
  -d '{"artist":"不存在的歌手"}'
```
响应:
```json
[]
```

## 数据库设计

### artists 表结构
```sql
CREATE TABLE IF NOT EXISTS artists (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255) NOT NULL UNIQUE COMMENT '歌手名称',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='歌手表';
```

### 数据来源
- 从 `music_files` 表中提取所有唯一歌手
- 自动处理合作歌手（如 "周杰伦&费玉清" 会被拆分为两个独立歌手）
- 当前已导入 15 位歌手

## 实现架构

### 代码结构
```
internal/artist/
├── model/artist.go                      # 歌手数据模型
├── repository/artist_repository.go      # 歌手数据访问层
├── service/artist_service.go            # 歌手业务逻辑层
└── handler/artist_handler.go            # 歌手HTTP处理层

internal/music/
├── model/music.go                       # 音乐数据模型
├── repository/music_repository.go       # 音乐数据访问层（包含 FindByArtist）
├── service/music_service.go             # 音乐业务逻辑层（包含 GetMusicByArtist）
└── handler/music_handler.go             # 音乐HTTP处理层（包含 GetMusicByArtist）
```

### 分层职责

#### 歌手搜索功能

1. **Model 层** (`artist/model/artist.go`)
   - 定义 Artist 实体
   - 定义请求/响应结构体

2. **Repository 层** (`artist/repository/artist_repository.go`)
   - `ExistsByName()`: 检查歌手是否存在（精确匹配）
   - `FindByName()`: 根据名称查找歌手

3. **Service 层** (`artist/service/artist_service.go`)
   - `SearchArtist()`: 搜索歌手业务逻辑
   - 参数验证和处理

4. **Handler 层** (`artist/handler/artist_handler.go`)
   - `SearchArtist()`: HTTP 请求处理
   - 请求解析和响应封装

#### 歌手音乐查询功能

1. **Repository 层** (`music/repository/music_repository.go`)
   - `FindByArtist()`: 使用模糊查询 (`LIKE`) 查找歌手的所有音乐

2. **Service 层** (`music/service/music_service.go`)
   - `GetMusicByArtist()`: 返回 `[]*FileListItem`，与 `GetAllMusic` 格式一致
   - 生成封面图片 URL

3. **Handler 层** (`music/handler/music_handler.go`)
   - `GetMusicByArtist()`: HTTP 请求处理
   - 参数验证，直接返回数组格式
