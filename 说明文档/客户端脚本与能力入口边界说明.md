# 客户端脚本与能力入口边界说明（当前版）

## 1. 文档目标
本文档用于明确：哪些能力适合直接开放给 Agent 或脚本，哪些能力应该继续停留在主窗口或人工交互层。

## 2. 当前适合开放的能力
- 搜索歌曲、搜索歌手、按歌手取歌；
- 查询当前播放、播放队列；
- 播放、暂停、恢复、停止、切歌、跳进度、设置音量；
- 查询和管理收藏、历史、歌单；
- 查询视频列表等低副作用信息能力。

## 3. 当前不宜直接开放或需更强保护的能力
- 欢迎页服务地址修改与模式切换；
- 离线直进与在线模式切换；
- 用户名修改、头像上传；
- 清空播放队列、批量删历史、批量改歌单等高副作用动作；
- 任何需要明确文件选择或本地路径确认的操作。

## 4. 当前版本的新增边界
由于本轮新增了离线本地模式、个人主页与下载元数据补偿，因此脚本和工具在描述能力时必须明确：
- 离线模式下不可用的在线动作；
- 用户资料链当前是主窗口直接协调，不是完全抽象成公共工具；
- 下载页元数据回写属于播放链补偿，不应被视为可单独操作的业务工具。
