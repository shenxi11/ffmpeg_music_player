# 客户端脚本化改造：阶段 1 记录（当前版归档）

## 1. 文档定位
本文档保留“阶段 1”作为历史记录，但内容已按当前代码校准，只记录当前仍然有效的阶段性结论，不再沿用已失效的设想。

## 2. 阶段 1 已完成的基础能力
- 客户端已具备 Agent 协议入口；
- 已具备工具声明、工具执行器和顺序脚本执行器三层；
- 已能把搜索、当前播放、队列、播放控制、收藏、历史、歌单等部分能力映射到真实业务入口；
- 客户端文档、能力映射和主窗口主链已能支撑后续继续扩展 Agent 能力。

## 3. 当前阶段 1 的真实边界
- 当前脚本 DSL 仍然是顺序执行；
- 不支持条件、循环、并发、事务回滚；
- Agent 不应直接控制欢迎页、用户资料页或离线模式切换，除非后续明确开放工具；
- 复杂对象解析和歧义消解仍建议放在服务端完成。

## 4. 阶段 1 对后续工作的价值
- 已证明客户端可以作为可编排执行端，而不是只能做被动 UI；
- 已形成“文档 -> 能力映射 -> 工具执行 -> 主业务链”的基础闭环；
- 为第二阶段继续扩展脚本执行语义和更细粒度能力打下基础。
