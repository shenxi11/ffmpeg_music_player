# 用户音乐功能测试脚本（当前版说明）

## 1. 文档目标
原测试脚本主要覆盖收藏与历史接口。当前版本在此基础上已经扩展出个人资料、歌单、欢迎页和离线模式等新链路，因此本文档改为“当前测试范围说明”，保留原脚本可作为接口回归样例，但不再把它视为完整测试集。

## 2. 原脚本仍可覆盖的内容
原脚本可继续用于验证：
- 收藏添加 / 删除 / 查询；
- 播放历史添加 / 查询；
- 历史接口基本可用性；
- 用户音乐基础数据格式是否符合当前服务端约定。

## 3. 当前版本新增必须人工补测的项
### 3.1 欢迎页
- `/client/ping` 与 `/client/bootstrap` 验证；
- 离线直进；
- 从设置页返回欢迎页并重新进入。

### 3.2 用户资料
- 登录后个人主页资料刷新；
- 用户名修改；
- 头像上传；
- 顶部头像与弹窗头像即时刷新。

### 3.3 播放与列表一致性
- 本地、下载、在线、历史、收藏列表的封面播放态；
- 播放队列封面和歌手补偿；
- 本地歌曲播放后 artist 持久化；
- 下载页播放后封面/歌手/时长回写。

### 3.4 歌单
- 创建歌单；
- 在线歌曲加歌单；
- 新建歌单并加入；
- 歌单详情查看与排序。

## 4. 当前建议
- 保留原测试脚本作为接口示例；
- 真正的回归测试应结合 UI 手测与当前主功能链分模块执行；
- 论文或验收材料里应说明：系统测试已从“收藏/历史接口验证”扩展到“启动、资料、播放、歌单、下载、视频”的综合测试。
