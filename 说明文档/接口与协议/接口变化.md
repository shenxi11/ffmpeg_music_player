# 接口变化（当前版汇总）

## 1. 文档目标
本文档用于汇总当前客户端已经接入、且相较于 2026-03-25 之前版本发生明显扩展的服务端接口能力。重点不是穷举所有历史接口，而是说明当前版本客户端真正依赖的接口面，以及近两周新增或强化的接口类别。

## 2. 当前接口分组

### 2.1 欢迎页与启动校验
当前欢迎页启动链依赖：
- `GET /client/ping`
- `GET /client/bootstrap`

作用：
- 校验服务可达性；
- 获取客户端启动所需的基础配置；
- 决定是否进入主窗口。

同时，客户端现已支持不走这两条接口而直接进入离线本地模式。

### 2.2 账号与资料
当前客户端已依赖以下资料相关接口：
- 登录
- 注册
- 重置密码
- `GET /users/profile`
- 用户名更新接口
- 头像上传接口

其中最近一轮版本变化的重点是：
1. 登录成功后，客户端会缓存 `avatar_url` 与 `online_session_token`；
2. 个人主页会主动拉取 `GET /users/profile`；
3. 用户名修改与头像上传已形成完整客户端链路。

### 2.3 收藏与最近播放
在原有读取能力之外，当前客户端已经接入：
- 批量删除历史记录；
- 清空历史记录；
- 收藏添加 / 删除；
- 最近播放写入与删除。

这些接口现在已经与主窗口、历史页、用户主页预览和播放写回链联动。

### 2.4 私有歌单
当前客户端已完整接入 playlist 相关接口：
- 获取歌单列表；
- 获取歌单详情；
- 创建歌单；
- 更新歌单信息；
- 删除歌单；
- 批量添加歌曲；
- 批量移除歌曲；
- 歌单排序。

最近版本中，一个关键修复是：
- 在线歌曲“加入歌单 / 新建歌单并加入”应保存 canonical path，而不是解析后的播放 URL。

### 2.5 推荐与歌手检索
当前客户端已接入：
- 推荐列表；
- 相似推荐；
- 推荐反馈；
- `POST /artist/search`；
- `POST /music/artist`。

歌手搜索能力现在既服务于搜索扩展，也服务于文档和论文中对“私域音乐检索”的能力描述。

### 2.6 视频
当前客户端已接入：
- 视频列表接口；
- 视频流地址接口。

视频链目前定位为“在线视频浏览和播放”，并不与音频链完全共用同一套 UI 容器。

## 3. 近两周最关键的接口变化
### 3.1 资料接口落地
这是 4 月 6 日之后最重要的新增能力之一：
- 当前客户端已经不再只显示静态用户名，而是支持资料刷新、用户名修改和头像上传；
- 顶部头像、用户弹窗和个人主页共享同一套资料缓存。

### 3.2 历史删除与清空正式纳入主链
历史列表不再只是“只读展示”：
- 可以删除选中记录；
- 可以清空全部历史；
- 当前 UI 与 `MainShellViewModel` 已联动这些接口。

### 3.3 歌手接口纳入搜索扩展面
歌手搜索和按歌手列歌功能已被客户端使用，不再只是服务端独立接口。

### 3.4 视频接口进入稳定展示链
视频接口已被 `VideoListViewModel` 和 `VideoPlayerWindow` 连接为当前版本可用链路。

## 4. 当前客户端视角下的接口清单（按 ViewModel 归属）
| ViewModel / 页面链 | 当前主要接口类别 |
|---|---|
| `ServerWelcomeViewModel` | `/client/ping`、`/client/bootstrap` |
| `LoginViewModel` | 登录、注册、重置密码 |
| `MainShellViewModel` | 推荐、相似推荐、反馈、收藏、历史、歌单、资料、歌手搜索 |
| `OnlineMusicListViewModel` | 在线搜索、流地址、在线下载、歌手音乐列表 |
| `VideoListViewModel` | 视频列表、视频流地址 |

## 5. 当前版本结论
1. 当前客户端接口面已从“登录 + 搜索 + 播放”扩展到“资料、歌单、推荐、历史、视频、歌手检索”一整套业务集；
2. 近两周接口变化最大的模块是：资料、歌单、历史、歌手检索；
3. 后续论文中描述系统功能时，应以本文档为“客户端已接入接口基线”，而不是引用旧的单接口说明。
