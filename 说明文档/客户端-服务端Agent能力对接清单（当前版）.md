# 客户端-服务端 Agent 能力对接清单（当前版补校准）

## 1. 文档定位
本文档仍然用于服务端 Agent 对接当前 Qt 客户端，但本次补校准后，强调一点：**Agent 能力是建立在当前客户端主链上的二次封装，而不是独立于主窗口、播放链、资料链之外的另一套系统。**

## 2. 当前仍然成立的基础判断
- `ToolRegistry` 负责声明工具；
- `AgentToolExecutor` 负责把工具映射到真实业务入口；
- `AgentScriptExecutor` 负责顺序脚本；
- 当前 DSL 仍然只支持顺序步骤，不支持复杂控制流。

## 3. 需要补充认知的当前版本变化
### 3.1 启动模式变化
客户端当前已支持：
- 欢迎页验证后进入；
- 离线直进进入。

因此服务端 Agent 在描述客户端能力时，应默认区分：
- 在线模式下可用的远程能力；
- 离线本地模式下不可用的能力。

### 3.2 用户资料链变化
客户端当前新增了个人主页、用户名修改、头像上传等主链能力。若未来 Agent 需要扩展资料相关工具，应优先复用：
- `MainShellViewModel`
- `UserProfileWidget`
- 当前 profile 缓存与刷新链

### 3.3 播放与列表状态变化
当前播放状态与列表封面动作已统一到 `PlaybackViewModel` + `SongCoverAction` 体系。Agent 若需要查询“当前播放”“当前队列”“当前歌曲封面/歌手/标题”，应优先依赖统一状态快照，而不是猜页面状态。

## 4. 当前最稳的对接原则
1. 先依赖当前主窗口和 ViewModel 的既有业务入口；
2. 不要让 Agent 直接假定所有页面都可在离线模式下使用；
3. 任何新增工具都应先确认当前代码里是否存在稳定业务入口，再决定是否暴露给 Agent。

## 5. 当前版本结论
这份对接清单仍然有效，但从 2026-04-06 起，Agent 对客户端的理解必须把“欢迎页模式、个人资料链、统一播放状态、下载与本地缓存补偿”一起纳入上下文，不能继续停留在旧版简单工具清单视角。
