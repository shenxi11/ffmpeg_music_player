# 私有歌单接口文档（当前版）

## 1. 文档目标
本文档描述当前客户端已经接入的私有歌单接口集合，以及客户端在 2026-04-08 版本下对歌单路径语义的实际要求。

## 2. 当前客户端使用场景
当前客户端中的歌单接口主要服务以下页面与动作：
- 我的歌单列表；
- 歌单详情页；
- 新建歌单；
- 编辑歌单信息；
- 删除歌单；
- 从列表页把歌曲加入歌单；
- 新建歌单并加入当前歌曲；
- 歌单内删除歌曲；
- 歌单拖拽排序。

## 3. 当前字段约定
### 3.1 歌单主体
- `id`：歌单 ID
- `name`：歌单名称
- `description`：歌单说明
- `cover_path` / `cover_url`：歌单封面
- `track_count`：歌曲数量
- `total_duration_sec`：总时长
- `created_at` / `updated_at`：创建时间和更新时间

### 3.2 歌单项
客户端当前会在 item 中保留：
- `music_path`：歌曲主标识
- `title`
- `artist`
- `cover_art_url`
- `duration`
- `is_local`

其中一个当前版本非常关键的约束是：
- **在线歌曲加入歌单时，必须保留 canonical path（原始业务路径），不能保存解析后的播放 URL。**

否则会导致：
- 歌单详情里出现一次性流地址；
- 重启或重新请求后路径失效；
- 客户端后续无法稳定识别该歌曲。

## 4. 当前接口集合
### 4.1 列表与详情
- `GET /user/playlists`
- `GET /user/playlists/{playlist_id}`

### 4.2 创建与更新
- `POST /user/playlists`
- 更新歌单信息接口
- 删除歌单接口

### 4.3 歌曲项管理
- 批量添加歌曲到歌单接口
- 批量移除歌单歌曲接口
- 歌单排序接口

## 5. 当前客户端接入要求
1. 用户账号沿用当前 profile 风格，通过 query 或 header 传递；
2. 本地歌曲和在线歌曲都允许进入歌单，但 `is_local` 必须与路径语义匹配；
3. 在线歌曲加歌单时不应经过播放 URL 解析链；
4. 歌单详情页显示时，应优先使用服务端返回的稳定字段，不在客户端二次猜测路径语义；
5. 当前歌单预览页和个人主页中的“创建的歌单”预览依赖这些接口返回的最新结果。

## 6. 当前版本结论
- 私有歌单已经从“规划能力”进入稳定业务主链；
- 歌单路径的 canonical 语义是当前版本最重要的接口约束之一；
- 后续论文描述歌单能力时，可以将其概括为“支持用户侧歌单全生命周期管理与本地/在线混合歌曲组织”。
