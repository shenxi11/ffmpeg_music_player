# 客户端开放接口清单（当前版）

## 1. 接口总说明

- 本文档以 `src/agent/tool/ToolRegistry.cpp` 和 `src/agent/tool/AgentToolExecutor.cpp` 作为当前版客户端开放接口的唯一真相来源。
- 这里的“开放接口”特指 **Qt 客户端对 Agent/自动化层开放的工具接口**，不包含服务端 HTTP API、QML 内部信号，也不包含历史 sidecar / websocket 协议。
- 登录 / 注册 / 重置密码、欢迎页 `client/ping` / `client/bootstrap`、离线直进模式进入链仍属于内部能力，不纳入开放接口。
- 当前版共开放 **87** 个工具接口；本轮在原有业务接口基础上，继续补齐了：
  - 在线音乐发起下载接口 `downloadTrack`
  - 页面级 UI 查询接口
  - 音乐 tab 统一查询与操作接口

### 3.12 UI 页面查询与音乐 Tab 统一接口说明

- 这组接口的设计目标是开放**页面级状态与当前可见业务数据**，而不是开放整个 QML 控件树。
- 统一 `page` 枚举：
  - `recommend`
  - `local_download`
  - `online_music`
  - `history`
  - `favorites`
  - `playlists`
  - `user_profile`
  - `video`
  - `settings`
- 统一 `tab` 枚举：
  - `recommend`
  - `local_music`

  - `online_music`
  - `history`
  - `favorites`
  - `playlist_list_owned`

### 3.14 UI 接口使用示例

以下示例统一采用“工具名 + 结构化参数”的调用形式，重点展示当前最常用的页面级查询和音乐 tab 操作方式。

#### 3.14.1 获取主窗口总览

用途：
- 判断当前主页面、离线模式、当前音乐 tab、当前歌单上下文。

调用示例：

```json
{
  "tool": "getUiOverview",
  "args": {}
}
```

返回重点：
- `currentPage`
- `offlineMode`
- `loggedIn`
- `currentMusicTab`
- `localDownloadSubTab`
- `sidebarPlaylistTab`
- `selectedPlaylist`
- `selectedTrackIds`

#### 3.14.2 获取指定主页面状态

用途：
- 查询某个主页面当前的摘要状态，而不是抓取整个控件树。

调用示例：

```json
{
  "tool": "getUiPageState",
  "args": {
    "page": "local_download"
  }
}
```

常用 `page`：
- `recommend`
- `local_download`
- `online_music`
- `history`
- `favorites`
- `playlists`
- `user_profile`
- `video`
- `settings`

#### 3.14.3 查询某个音乐 tab 的当前可见条目

用途：
- 读取当前页面已经加载出来的业务数据，例如本地音乐、下载列表、歌单详情歌曲等。

调用示例：

```json
{
  "tool": "getMusicTabItems",
  "args": {
    "tab": "local_music",
    "limit": 20
  }
}
```

歌单详情示例：

```json
{
  "tool": "getMusicTabItems",
  "args": {
    "tab": "playlist_detail",
    "playlistId": "123456"
  }
}
```

说明：
- 当前实现以“当前页面可见数据”为真相。
- 对 `playlist_detail`，如果当前没有歌单详情上下文，会返回：
  - `playlist_context_required`

#### 3.14.4 按业务标识读取单条歌曲或歌单对象

用途：
- 在统一 tab 里按 `trackId / musicPath / playlistId / playlistName` 定位一条对象。

调用示例：

```json
{
  "tool": "getMusicTabItem",
  "args": {
    "tab": "history",

```

歌单列表示例：

```json
{

  "args": {
    "tab": "playlist_list_owned",
    "playlistName": "流行歌单"
  }
}
```

#### 3.14.5 在指定音乐 tab 中播放一首歌

用途：
- 用统一接口从推荐、本地、历史、歌单详情等 tab 里播放一首歌。

调用示例：

```json
{
  "tool": "playMusicTabTrack",
  "args": {
    "tab": "playlist_detail",
    "trackId": "track_001",
    "playlistName": "流行歌单"
  }
}
```

返回重点：
- `tab`
- `played`
- `item`
- `result`

常见错误：
- `tab_item_not_found`
- `playlist_context_required`
- `unsupported_action_for_tab`

#### 3.14.6 对音乐 tab 执行统一业务动作

用途：
- 对不同 tab 中的歌曲执行稳定业务动作，而不是模拟点击某个按钮。

统一 `action`：
- `toggle_current_playback`
- `play_next`
- `add_to_playlist`
- `download`
- `add_favorite`
- `remove_favorite`
- `remove_or_delete`

在线歌曲下载示例：

```json
{
  "tool": "invokeMusicTabAction",
  "args": {
    "tab": "online_music",
    "action": "download",
    "trackId": "track_online_001"
  }
}
```

从本地音乐页删除歌曲示例：

```json
{
  "tool": "invokeMusicTabAction",
  "args": {
    "tab": "local_music",
    "action": "remove_or_delete",
    "musicPath": "E:/Music/周杰伦/七里香.mp3"
  }
}
```

向歌单加歌示例：

```json
{
  "tool": "invokeMusicTabAction",
  "args": {
    "tab": "recommend",
    "action": "add_to_playlist",
    "trackId": "track_online_002",
    "playlistName": "流行歌单"
  }
}
```

#### 3.14.7 tab 与 action 支持边界

- `recommend` / `online_music`
  - 支持：播放、切换当前播放、下一首、加歌单、下载、收藏、取消收藏
- `local_music` / `downloaded_music`
  - 支持：播放、切换当前播放、下一首、加歌单、收藏、取消收藏、删除/移除
- `history`
  - 支持：播放、切换当前播放、下一首、收藏、取消收藏、删除/移除
- `favorites`
  - 支持：播放、切换当前播放、下一首、取消收藏
- `playlist_detail`
  - 支持：播放、切换当前播放、下一首、加歌单、下载、收藏、取消收藏、删除/移除
- `downloading_tasks`
  - 不走歌曲动作接口，应使用下载任务工具：
    - `pauseDownloadTask`
    - `resumeDownloadTask`
    - `cancelDownloadTask`
    - `removeDownloadTask`
- `playlist_list_owned` / `playlist_list_subscribed`
  - 主要用于查询歌单对象
  - 若要操作歌单内歌曲，应先进入 `playlist_detail`

#### 3.14.8 按功能场景分组的典型调用

下面这些场景示例更接近真实联调顺序，可以直接作为 Agent、自动化脚本或论文中的“接口调用流程”参考。

##### 场景 A：查看当前页面与当前音乐 tab

目标：
- 判断用户当前停留在哪个主页面、哪个音乐 tab，以及当前是否处于离线模式。

推荐调用顺序：

```json
{
  "tool": "getUiOverview",
  "args": {}
}
```

典型用途：
- 先判断 `currentPage`
- 再判断 `currentMusicTab` / `localDownloadSubTab`
- 再决定后续读哪个 tab 数据

##### 场景 B：读取当前歌单里的歌曲

目标：
- 在“我的歌单”或歌单详情页里读取当前歌单的歌曲列表。

推荐调用顺序：

```json
{
  "tool": "getUiOverview",
  "args": {}
}
```

确认 `selectedPlaylist` 或当前位于 `playlist_detail` 后，再调用：

```json
{
  "tool": "getMusicTabItems",
  "args": {
    "tab": "playlist_detail",
    "playlistId": "123456"
  }
}
```

如果只知道歌单名，也可以用：

```json
{
  "tool": "getMusicTabItems",
  "args": {
    "tab": "playlist_detail",
    "playlistName": "流行歌单"
  }
}
```

当前没有歌单详情上下文时，预期错误：
- `playlist_context_required`

##### 场景 C：从歌单页播放某首歌

目标：
- 在歌单详情页中，按业务标识播放一首歌。

推荐调用顺序：
1. 先 `getMusicTabItems(tab=playlist_detail)`
2. 再根据返回条目里的 `trackId` 或 `musicPath` 调用播放

播放示例：

```json
{

  "args": {
    "tab": "playlist_detail",
    "trackId": "track_001",
    "playlistName": "流行歌单"
  }
}
```

##### 场景 D：读取本地音乐并播放其中一首

目标：
- 查询本地音乐列表，然后播放其中一首。

第一步，读列表：

```json

  "tool": "getMusicTabItems",
  "args": {
    "tab": "local_music",
    "limit": 50
  }
```

第二步，按 `musicPath` 或 `trackId` 播放：

```json
{

  "args": {
    "tab": "local_music",
    "musicPath": "E:/Music/周杰伦/七里香.mp3"
  }
}
```

##### 场景 E：从在线音乐列表下载一首歌

目标：
- 在在线音乐页中，对某首在线歌曲发起下载。

推荐调用顺序：
1. `getMusicTabItems(tab=online_music)`
2. 选定目标条目
3. `invokeMusicTabAction(action=download)`

调用示例：

```json
{
  "tool": "invokeMusicTabAction",
  "args": {
    "tab": "online_music",
    "action": "download",
```

##### 场景 G：从喜欢音乐页取消喜欢

目标：

调用示例：

```json
}
```

说明：
- `favorites` 页不支持 `remove_or_delete`
- 如果误调用，会返回：

##### 场景 H：查询下载页三个子 tab 的内容

目标：

  - `resumeDownloadTask`
  - `cancelDownloadTask`
  - `removeDownloadTask`

##### 场景 I：先找歌单，再进入歌单详情

## 4. 未开放 / 保持内部的能力说明

### 4.1 保持内部的能力
- 登录 / 注册 / 重置密码：属于认证入口，不开放为 Agent 工具。
- 欢迎页 `client/ping` / `client/bootstrap`：属于客户端启动与服务验证链，只作为内部 bootstrap 使用。
- 欢迎页进入在线模式 / 离线直进：属于模式切换入口，不作为普通业务工具开放。
- 仅用于页面视觉控制、没有稳定业务语义的 QML 内部信号：不纳入工具接口。
- 基于 UI 行列索引的行级/按钮级定位：过于脆弱，不作为正式接口真相开放。
- 任意 QML 控件树直接操作：会和页面实现细节强耦合，保持内部。

### 4.2 历史兼容层说明
- 历史 sidecar / websocket / `8765` 相关链路不再是桌面主链真相，也不应作为“客户端开放接口”来源。
- `agentLocalModelBaseUrl` 作为兼容字段仍然存在，但不代表当前桌面主链仍依赖 HTTP 模型服务。

### 4.3 审计结论
- 以 `说明文档/当前客户端功能与实现层级树.md` 为基准，当前客户端主要功能已全部落到“已开放接口”或“内部能力”之一。
- 本轮确认并补齐的明确缺口包括：
  - `downloadTrack`：把在线音乐“发起下载”能力纳入开放接口面
  - `getUiOverview / getUiPageState`：把页面级 UI 状态摘要纳入开放接口面
  - `getMusicTabItems / getMusicTabItem / playMusicTabTrack / invokeMusicTabAction`：把音乐 tab 的统一查询与业务动作纳入开放接口面
- 后续若功能树新增一级能力，应同步以 `ToolRegistry + AgentToolExecutor` 为真相重新审计，而不是继续扩写旧主题文档。
