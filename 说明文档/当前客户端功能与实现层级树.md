从# 当前客户端功能与实现层级树

## 1. 文档目的
本文档用于从“用户功能视角”和“代码实现视角”同时梳理当前客户端的真实结构，作为后续说明文档补充、PPT 架构图输入和毕业论文系统结构章节的直接底稿。

本文档只描述当前版本主链，不把历史原型、兼容链路或已降级的旧 sidecar 运行路径当成当前真相。

## 4. 当前主链模块说明

### 4.1 桌面运行主链
当前桌面主链是：
- `main.cpp -> ServerWelcomeDialog -> MainWidget`
- `MainWidget` 负责壳层、页面装配、模式控制与全局连接
- `MainShellViewModel` 负责账号、资料、歌单、推荐、历史、收藏、下载等业务聚合
- `PlaybackViewModel` 负责播放状态统一输出

### 4.2 页面与状态分工
当前客户端采用 Widgets + QML 混合组织：
- `src/ui/widgets` 负责页面宿主；
- `qml/components` 负责表现层和局部交互；
- `src/viewmodels` 负责业务状态和请求入口；
- `src/common/src/network/src/download` 等负责底层能力。

### 4.3 Agent 当前定位
当前 `src/agent` 已经属于客户端主功能之一，但其主链真相已经切到：
- `AgentChatViewModel -> AgentLocalRuntime -> AgentLocalModelGateway -> AgentEmbeddedLlamaEngine`
- 页面级 UI 查询与音乐 tab 操作接口当前统一通过：
  - `ToolRegistry + AgentToolExecutor -> MainWidget UI 门面快照`
- 旧 `8765/websocket/http sidecar` 只作为兼容与历史参考保留，不再是桌面运行时前提。

## 5. 兼容层与历史残留说明
以下内容仍可保留在仓库中，但不应再作为当前客户端主链来理解：

- `src/agent/AgentProcessManager.*`
- `src/agent/AgentWebSocketClient.*`
- `agent/src/music_agent/server.py` 及其 websocket / sidecar 协议链
- 仓库中历史原型或早期旧页面路径
- 以 `8765`、`/healthz`、sidecar attach 为前提的旧 Agent 运行理解

这些内容当前的正确定位是：
- 历史实现
- 兼容参考
- 开发对照

而不是桌面客户端当前版本的运行真相。

## 6. 可直接复用的结论
如果后续要把当前软件结构写成 PPT 或论文摘要，可以直接提炼为以下结论：

1. 客户端当前已经形成“主窗口壳层 + ViewModel 业务聚合 + Widgets/QML 混合页面 + 核心能力层”的清晰分层；
2. `MainShellViewModel` 和 `PlaybackViewModel` 分别承担业务状态轴与播放状态轴；
3. 本地、下载、歌单、推荐、用户资料、视频、设置与 AI 助手都已纳入当前统一客户端结构；
4. AI 助手已从外部 sidecar 模式逐步收敛到客户端内部主链，不再只是外部附属功能。
