# AI 理解软件运行逻辑说明（当前版）

## 1. 文档目的
本文档面向两类读者：
- 需要快速理解当前 Qt 客户端如何运行、如何联动的开发者；
- 后续需要基于当前实现撰写论文、总结系统设计或继续扩展 Agent 能力的辅助写作者。

本文档不再只从 Agent 角度解释系统，而是先说明当前客户端真实主链，再补充 Agent 如何接入这些主链。

## 2. 当前软件的运行入口
### 2.1 启动入口
应用启动统一从 `src/app/main.cpp` 进入，当前逻辑分为两种模式：
1. 通过 `ServerWelcomeDialog` 校验服务器可用性后进入主窗口；
2. 通过欢迎页的“`不连接，直接进入`”直接进入离线本地模式。

主窗口统一由 `MainWidget(bool localOnlyMode)` 构造，因此“在线模式”和“离线本地模式”并不是两套客户端，而是同一主窗口的两种运行形态。

### 2.2 主窗口职责
`MainWidget` 是当前系统的壳层协调器，负责：
- 页面装配与切换；
- 在线导航与离线模式 UI 收口；
- 搜索框、用户入口、列表页、播放页、设置页的全局连接；
- 资料刷新、头像上传、用户名修改、欢迎页返回等跨页面流程；
- 将 `PlayWidget` 的播放元数据回写到本地缓存与下载模型。

## 3. 当前软件有哪些可控能力
如果从“AI 或自动化工具能理解并调用什么”这个角度看，当前客户端能力可概括为：

### 3.1 本地与下载
- 本地音乐扫描与展示；
- 下载任务展示、暂停/恢复、已下载列表管理；
- 本地播放、播放控制、播放详情页；
- 播放后把封面、时长、歌手回写到本地缓存。

### 3.2 在线音乐
- 搜索音乐；
- 推荐音乐、相似推荐；
- 在线流播放；
- 收藏、历史、加入歌单、新建歌单并加入；
- 在线音乐下载。

### 3.3 用户资料
- 登录 / 注册 / 重置密码；
- 用户资料刷新；
- 用户名修改；
- 头像上传与刷新；
- 个人主页内容预览（喜欢、创建的歌单、最近播放）。

### 3.4 视频
- 视频列表获取；
- 在线视频流地址解析；
- 视频窗口播放。

### 3.5 欢迎页与模式切换
- 服务地址验证；
- 服务 bootstrap；
- 离线直进；
- 从主窗口返回欢迎页后重新选择模式。

## 4. 当前软件状态是如何被统一管理的
### 4.1 业务状态
业务状态主要由 `MainShellViewModel` 承担，典型内容包括：
- 账号和资料缓存；
- 推荐、收藏、历史、歌单；
- 资料刷新与更新结果；
- 本地缓存回写入口；
- 下载与播放相关的业务补偿。

### 4.2 播放状态
播放状态统一由 `PlaybackViewModel` 管理：
- 当前标题、歌手、封面；
- 是否播放、播放模式、进度、时长；
- 当前 URL / 文件路径；
- `playlistSnapshot` 队列快照。

这意味着列表页、播放页、播放队列、封面覆盖层等都不应再各自维护一套状态源。

### 4.3 页面状态
QML 页面只负责表现和局部交互，复杂业务状态一般通过：
- `QQuickWidget` 宿主；
- QML bridge；
- ViewModel 信号/属性；
完成同步。

## 5. 当前关键调用链
### 5.1 搜索与在线列表
- 用户在搜索框输入关键字；
- `MainWidget` 转发到 `MainShellViewModel` 或在线列表相关 ViewModel；
- `HttpRequestV2` 发起请求；
- 结果回到在线列表、推荐页或其它容器；
- 用户再从列表发起播放、下载、收藏或加歌单动作。

### 5.2 播放与元数据补偿
- 列表页或播放页发起播放；
- `PlaybackViewModel` 驱动 `AudioService`；
- 播放期间若解析到封面、歌手、时长，`PlayWidget` 统一发出 `signalMetadataUpdated(...)`；
- `MainWidget` 将其回写到本地列表、下载列表和本地缓存；
- 下次打开应用时，这些歌曲仍可展示真实封面和歌手。

### 5.3 资料链
- 顶部头像入口进入用户弹窗或个人主页；
- 个人主页发起资料刷新、用户名修改、头像上传；
- `MainShellViewModel` 调用 `HttpRequestV2`；
- `SettingsManager` 更新缓存；
- 顶部头像、弹窗头像、个人主页头像同步刷新。

### 5.4 离线模式
- 欢迎页选择离线直进；
- 主窗口隐藏在线导航并禁用搜索；
- 本地与下载、本地播放、设置页仍可使用；
- 所有服务端相关入口会在业务层再次被拦截。

## 6. Agent 在当前系统中的位置
当前仓库中仍保留 `src/agent` 目录。Agent 的定位不是直接操作控件，而是：
- 通过协议层收发工具调用；
- 通过 `HostStateProvider` 获取统一状态快照；
- 通过 `AgentToolExecutor` 映射到已有业务入口。

也就是说，Agent 是“系统外部的协议驱动层”，而不是一条独立于主窗口、播放链、资料链之外的新业务主线。

## 7. 当前版本最值得记住的结论
1. 系统的真实入口是 `main.cpp -> ServerWelcomeDialog -> MainWidget`；
2. `MainShellViewModel` 是业务聚合中心，`PlaybackViewModel` 是播放状态中心；
3. 本地、下载、在线、历史、个人资料都已被连接到统一状态链，而不是各自孤立；
4. 当前版本最大的增量来自 4 月 6 日之后：播放状态统一、用户主页、欢迎页离线直进、下载补封面与本地歌手持久化。
