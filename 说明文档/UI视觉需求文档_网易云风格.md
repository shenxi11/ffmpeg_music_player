# UI 视觉需求文档（当前版）

## 1. 文档目标
本文档用于固化当前客户端的视觉基线。虽然整体仍以“网易云风格的轻桌面播放器”为主，但 2026-04-06 之后的改动已经引入部分更接近 QQ 音乐内容页的设计语言，因此本文档改为描述**当前实际视觉结果**，不再把所有页面都强行归为单一旧风格。

## 2. 当前视觉基线
### 2.1 总体方向
- 主体仍是浅色桌面音乐客户端；
- 红色强调继续承担品牌动作色和播放控制强调色；
- 列表、控制栏、播放页延续“轻边框 + 细圆角 + 红色点缀”的网易云式气质；
- 用户主页采用更接近 QQ 音乐的内容流布局：大头像、轻标签、内容预览主导。

### 2.2 当前配色原则
- 主强调色：`#EC4141`
- 强调 hover：更浅一档的红色或柔和红底
- 主背景：白色或接近白色的浅灰底
- 主要文字：深色
- 次级文字：灰蓝色
- 分割线与描边：浅灰蓝色

### 2.3 当前圆角与组件语言
- 主列表行：轻圆角
- 弹窗与卡片：中等圆角
- 主控制按钮：圆形或胶囊形
- 用户主页头部按钮：轻胶囊、小描边、低对比度
- 个人主页大头像：圆形裁切

## 3. 当前页面级视觉要求
### 3.1 欢迎页
- 以轻表单卡片为主；
- 右下角主按钮承担主要动作；
- “不连接，直接进入”采用可点击文本，而不是厚重次按钮。

### 3.2 主窗口与列表页
- 左侧导航保持稳定的纵向信息架构；
- 列表行悬浮时显示统一封面播放态与动作条；
- 本地、下载、在线、收藏、历史列表的封面交互保持一致。

### 3.3 播放页
- 播放详情页采用统一舞台布局；
- 左侧唱片/封面与右侧歌词卡片共享基线；
- 主控制栏图标保持统一免费线性风格；
- 视觉重点从“装饰”转向“播放状态清晰 + 布局平衡”。

### 3.4 个人主页
- 当前个人主页不再使用厚重统计卡；
- 采用内容流布局：资料头部 + 标签页 + 内容预览；
- 资料页按钮应保持轻、细、克制；
- 头像上传与用户名修改是头部轻入口，而不是后台表单式布局。

## 4. 当前资源约束
- 图标资源优先使用统一 SVG；
- 默认封面、默认头像、主控制栏图标和列表动作图标要尽量保持风格一致；
- 允许不同页面存在轻微风格差异，但禁止同一页面中混用明显不属于同一套的图标语言。

## 5. 当前版本结论
后续所有 UI 调整都应以“当前真实界面”为基线：
- 主播放与列表链路偏网易云式；
- 用户主页偏 QQ 音乐式内容流；
- 整体原则是轻量、克制、统一，而不是追求单一品牌完全复刻。
