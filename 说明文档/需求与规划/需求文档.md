# 需求文档（当前版基线）

## 1. 文档目标
本文档用于固化 2026-04-08 时点客户端的当前功能边界与版本基线，供后续开发协作、论文撰写和功能回归使用。本文档不再记录单一接口变更，而是描述当前版本已经落地的整体需求集合。

## 2. 产品定位
当前项目是一个面向私域部署场景的桌面云音乐客户端，采用 Qt Widgets + QML 混合架构，支持：
- 本地音乐播放与管理；
- 私域服务器音乐搜索、在线播放与下载；
- 收藏、最近播放、私有歌单；
- 视频列表与视频播放；
- 用户登录、资料刷新、用户名修改与头像上传；
- 欢迎页服务验证与离线本地模式；
- Agent/脚本化扩展能力预留。

## 3. 当前版本核心需求

### 3.1 启动与首次进入
1. 客户端启动时必须先进入欢迎页；
2. 用户可输入服务器地址并验证 `/client/ping` 与 `/client/bootstrap`；
3. 验证通过后进入主窗口；
4. 用户也可通过“`不连接，直接进入`”进入离线本地模式；
5. 从设置页返回欢迎页后，应允许重新选择在线进入或离线直进。

### 3.2 本地与下载
1. 支持本地音乐展示与播放；
2. 支持下载音乐列表与正在下载列表；
3. 下载完成歌曲应支持封面、时长、歌手展示；
4. 当播放过程中提取到本地歌曲元数据时，应写回本地缓存，解决“未知艺术家”与默认封面长期不更新的问题；
5. 下载页与本地页的封面播放交互应保持统一。

### 3.3 在线音乐
1. 支持搜索、推荐、相似推荐；
2. 支持在线播放和流地址解析；
3. 支持收藏、最近播放、加歌单、新建歌单并加入、下载；
4. 在线列表、收藏列表、历史列表、歌单详情页应共享统一封面动作与播放态同步。

### 3.4 播放体验
1. 底部控制栏与播放详情页应共享统一播放状态；
2. 播放页支持多种风格，并以统一舞台布局管理唱片区和歌词区；
3. 右侧播放队列应与主播放状态同步，切歌后上一首封面和歌手信息不能丢失；
4. 当前播放、暂停、悬浮封面时的图标状态在主要列表页应保持一致。

### 3.5 用户与资料
1. 支持登录、注册、密码重置；
2. 顶部头像入口支持显示用户信息与弹出菜单；
3. 个人主页作为主内容区页面存在，不再只是小弹窗；
4. 个人主页支持：
   - 资料刷新
   - 用户名修改
   - 头像上传与更新
   - 喜欢 / 创建的歌单 / 最近播放预览
5. 用户头像上传成功后，个人主页、顶部头像、用户弹窗应立即刷新。

### 3.6 离线本地模式
1. 离线模式只保留本地与下载、本地播放、设置页；
2. 推荐、在线音乐、最近播放、我喜欢的音乐、歌单、视频、账号入口、搜索等服务器能力不可用；
3. 主窗口和业务入口层都要有离线模式保护，不允许残余调用链继续发起网络请求。

### 3.7 视频
1. 支持视频列表获取；
2. 支持根据路径获取视频流地址；
3. 支持打开视频播放窗口完成基本播放控制。

## 4. 非功能需求
1. **状态一致性**：播放态、队列态、封面态、资料态应尽量收敛到统一状态源，避免列表页各自维护独立真相；
2. **缓存持久化**：本地元数据、下载封面、用户资料缓存应在重启后保留；
3. **离线可用性**：即使服务器不可用，客户端仍应支持本地模式进入；
4. **可维护性**：主窗口、播放链、网络链、用户资料链都要保留清晰的 ViewModel 边界，便于后续扩展和论文总结。

## 5. 当前版本重点变化（相对 3 月版）
- 欢迎页新增离线直进；
- 播放页布局与控制栏图标统一重构；
- 用户主页从简易卡片升级为独立主页面；
- 列表封面交互统一组件化；
- 下载页与本地页补齐元数据回写；
- 本地歌曲播放后可持久化歌手信息；
- 队列、历史、在线列表的播放态与封面同步进一步收敛。

## 6. 当前不在本轮需求范围内的内容
- 社交动态与陌生人主页；
- 更复杂的视频管理后台；
- 多终端同步；
- 完整自动化 Agent 自主控制客户端。
