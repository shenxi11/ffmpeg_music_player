# 服务端远程开发发布工具说明

更新时间：2026-04-16

## 目标

把客户端仓库中的本地辅助脚本固定下来，后续可以直接从当前 Windows 机器远程读写虚拟机上的服务端代码，并触发正式部署。

当前约定：

- 虚拟机地址：`192.168.1.208`
- 用户：`shen`
- 开发测试目录：`/home/shen/microservice-deploy`
- 正式部署目录：`/home/shen/CloudMusic`
- 正式部署方式：在 `CloudMusic` 执行 `./scripts/deploy_from_main.sh`

## 脚本位置

- 远程工具脚本：`scripts/cloudmusic_remote.py`

该脚本默认支持以下能力：

- `probe`
  - 检查远端主机、两个仓库的 Git 状态和 Docker 容器状态
- `exec`
  - 在远端执行任意命令
- `read`
  - 读取远端文件到本地标准输出
- `upload`
  - 上传本地文件到远端
- `download`
  - 下载远端文件到本地
- `deploy`
  - 在正式目录执行 `./scripts/deploy_from_main.sh`

## 认证方式

脚本认证顺序：

1. 优先尝试专用私钥 `~/.ssh/id_ed25519_cloudmusic_vm`
2. 如果 key 认证失败，回退到密码
3. 密码优先读取环境变量 `CLOUDMUSIC_VM_PASSWORD`
4. 若未设置环境变量且当前终端可交互，则提示手动输入密码

推荐在当前 PowerShell 会话中临时设置密码，而不是写进脚本或仓库文件：

```powershell
$env:CLOUDMUSIC_VM_PASSWORD = '你的密码'
```

## 常用命令

探测远端整体状态：

```powershell
pwsh -NoLogo -NoProfile -Command "python scripts/cloudmusic_remote.py probe"
```

查看开发目录 Git 状态：

```powershell
pwsh -NoLogo -NoProfile -Command "python scripts/cloudmusic_remote.py exec --repo dev -- git status --short --branch"
```

读取正式目录配置：

```powershell
pwsh -NoLogo -NoProfile -Command "python scripts/cloudmusic_remote.py read --repo prod config.yaml"
```

上传本地文件到开发目录：

```powershell
pwsh -NoLogo -NoProfile -Command "python scripts/cloudmusic_remote.py upload --repo dev .\\local-file.txt configs/local-file.txt"
```

下载远端日志到本地：

```powershell
pwsh -NoLogo -NoProfile -Command "python scripts/cloudmusic_remote.py download --repo prod logs/gateway.log .\\tmp\\gateway.log"
```

执行正式部署：

```powershell
pwsh -NoLogo -NoProfile -Command "python scripts/cloudmusic_remote.py deploy"
```

## 推荐工作流

1. 在 `microservice-deploy` 目录改代码、验证和提交
2. 把变更合并到远端仓库的 `main`
3. 在 `CloudMusic` 目录执行正式部署脚本
4. 用 `probe` 再次检查容器和仓库状态

## 注意事项

- `upload` 与 `download` 的路径参数在指定 `--repo` 时可以传相对路径；未指定 `--repo` 时必须使用绝对路径
- 指定 `--repo` 时，即使在 Windows 里写了反斜杠，脚本也会自动转成 Linux 路径分隔符
- `deploy` 不会跳过正式目录内置的分支和工作区检查；若 `CloudMusic` 目录不干净，部署会被拒绝
- 若未来虚拟机完全放通新 key，可直接不设置 `CLOUDMUSIC_VM_PASSWORD`
