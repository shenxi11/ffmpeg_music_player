# 客户端网络联调清单（私域部署，当前版）

## 1. 文档目标
本文档用于指导当前版本客户端与私域服务端进行联调，重点覆盖欢迎页验证、账号资料、在线音乐、歌单、视频和离线直进场景。

## 2. 联调前置检查
### 2.1 服务端地址
欢迎页当前要求可访问的服务地址应填写“客户端所在机器可访问到的业务网卡地址”，而不是：
- `127.0.0.1`（除非服务就跑在同一台机器）；
- Docker 网桥地址，如 `172.17.x.x`、`172.18.x.x`；
- 仅虚拟机内部可见的桥接或容器地址。

### 2.2 欢迎页验证接口
欢迎页成功进入主窗口依赖：
- `GET /client/ping`
- `GET /client/bootstrap`

若出现 `Connection refused`，优先检查：
- 服务实际监听的网卡和端口；
- 是否填错成 Docker 网桥地址；
- 宿主机对虚拟机业务 IP 是否可达。

### 2.3 离线直进
即使欢迎页联调失败，当前版本仍可通过“`不连接，直接进入`”进入离线本地模式，以继续验证本地播放和设置链。

## 3. 当前联调优先顺序
建议按以下顺序联调：
1. 欢迎页验证；
2. 登录 / 注册 / 重置密码；
3. 搜索 / 推荐 / 在线列表；
4. 收藏 / 历史 / 歌单；
5. 用户资料（profile / 用户名 / 头像）；
6. 视频列表与视频播放；
7. 下载音乐与已下载列表补元数据。

## 4. 当前关键接口清单
### 4.1 启动与资料
- `/client/ping`
- `/client/bootstrap`
- 登录 / 注册 / 重置密码接口
- `GET /users/profile`
- 用户名更新接口
- 头像上传接口

### 4.2 音乐与歌单
- 搜索接口
- 推荐 / 相似推荐 / 反馈接口
- 收藏增删查接口
- 播放历史增删查 / 清空接口
- 歌单列表 / 详情 / 创建 / 更新 / 删除 / 批量加歌 / 批量删歌 / 排序接口
- `POST /artist/search`
- `POST /music/artist`

### 4.3 视频
- 视频列表接口
- 视频流地址接口

## 5. 当前联调重点验证项
### 5.1 用户资料
- 登录成功后是否能拿到 `avatar_url` 与 `online_session_token`；
- 个人主页是否能刷新 profile；
- 用户名修改后顶部头像区和个人主页是否同步；
- 上传头像后顶部头像、用户弹窗和个人主页是否立即刷新。

### 5.2 播放与列表
- 在线列表、历史列表、下载列表、喜欢列表是否能正确命中当前播放态；
- 播放队列封面、歌手和标题是否随切歌保持一致；
- 下载页播放后是否能把封面、时长、歌手回写到列表并持久化。

### 5.3 离线模式
- 离线直进后默认是否停在“本地与下载”；
- 在线导航是否隐藏；
- 搜索框是否禁用；
- 服务端动作是否被业务层拦截。

## 6. 当前推荐联调命令
以 Windows 宿主机联调为例：

```powershell
Invoke-WebRequest -UseBasicParsing -Uri "http://<host>:8080/client/ping"
Invoke-WebRequest -UseBasicParsing -Uri "http://<host>:8080/client/bootstrap"
```

服务端机上建议核对：

```bash
ip addr
ss -lntp | grep 8080
```

重点确认：
- 服务监听在 `0.0.0.0:8080` 或业务网卡地址；
- 客户端填写的是业务 IP，而不是 Docker 网桥 IP。

## 7. 当前版本结论
- 私域联调的第一障碍通常不是代码，而是服务地址选择；
- 当前客户端欢迎页、资料页、歌单页、视频页都已依赖更多服务端能力，联调时应按模块顺序推进；
- 离线直进可作为网络联调失败时的本地验证兜底。
