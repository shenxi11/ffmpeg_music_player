```cpp
void MediaSession::demuxLoop()
{
    while (m_demuxRunning) {
        int ret = av_read_frame(m_formatContext, pkt);
        if (pkt->stream_index == m_audioStreamIndex && m_audioCodecCtx) {
            // 音频：解码+滤镜+写入 AudioPlayer
            AudioPlayer::instance().writeAudioData(..., m_audioWriteOwnerId);
        } else if (pkt->stream_index == m_videoStreamIndex && m_videoSession) {
            AVPacket* clonedPkt = av_packet_clone(pkt);
            m_videoSession->pushPacket(clonedPkt);
        }
    }
}
```
作用：单线程读取容器包，音频走 `AudioPlayer`，视频包分发到 `VideoSession`。

### 5.2 seek 同步
代码：`MediaSession.cpp:335`
```cpp
void MediaSession::seekTo(qint64 positionMs)
{
    av_seek_frame(...);
    if (m_audioCodecCtx) {
        avcodec_flush_buffers(m_audioCodecCtx);
        AudioPlayer::instance().resetBuffer();
        m_seekPending = true;
    }
    if (m_videoSession) {
        m_videoSession->flush();
        QMetaObject::invokeMethod(m_videoSession->videoRenderer(), "startBuffering");
    }
    m_masterClock = positionMs;
}
```
作用：seek 后同时清空音视频缓冲，让新时间轴从目标位置重新建立。

## 7. 当前架构特征与改造建议
### 7.1 已有优势
- 视频链路模块边界清晰：`MediaSession`（编排）+ `VideoSession`（视频子系统）+ `VideoRendererGL`（渲染）。
- 同步策略明确：音频主时钟 + 视频追帧/等待策略。
- 支持质量预设和显示模式（Fit/Fill），便于 UI 侧扩展。

### 7.2 当前需要注意的点
- `MediaService` 与 `VideoPlayerWindow` 存在并行入口，实际播放主要走后者直建会话，后续可统一入口减少状态分叉。
- `MediaSession::syncAudioVideo()` 中视频位置比对仍有 TODO（当前偏向渲染器侧同步）。
- `VideoRenderer`（QWidget 版本）已基本被 `VideoRendererGL` 取代，可考虑逐步下线旧路径。

## 8. 排障优先定位建议
- 视频比例失真：先看 `VideoSession.cpp:83`（SAR读取）和 `VideoRendererGL.cpp:630`（PAR应用）、`VideoRendererGL.cpp:649`（Fit/Fill）。
- seek 后黑屏/卡住：先看 `MediaSession.cpp:335`（flush+startBuffering）与 `VideoRendererGL.cpp:708`（buffering门槛）。
- 音画不同步：先看 `MediaSession.cpp:450`（主时钟来源）和 `VideoRendererGL.cpp:708`（±100/200ms 判定）。
- 倍速异常：先看 `MediaSession.cpp:408` 和 `MediaSession.cpp:886` 的滤镜链重建日志。
