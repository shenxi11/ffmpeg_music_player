# 视频架构与内部数据同步（当前版，含关键代码说明）

## 1. 文档目标
本文档用于描述当前版本视频播放模块的真实结构，而不是保留旧版视频草案。重点关注：
- 视频播放当前实际运行路径；
- `MediaSession` 与 `VideoSession` 的职责边界；
- 音视频状态、进度和渲染链如何同步到窗口。

## 2. 当前视频模块结构
| 模块 | 代码位置 | 职责 |
|---|---|---|
| `VideoListViewModel` | `src/viewmodels/VideoListViewModel.*` | 获取视频列表与视频流 URL |
| `VideoListWidget` | `src/ui/widgets/video/video_list_widget.cpp` | 视频列表 UI 入口，负责打开播放窗口 |
| `VideoPlayerWindow` | `src/video/VideoPlayerWindow.*` | 当前版本视频播放窗口、控制条、倍速和清晰度入口 |
| `MediaService` | `src/video/MediaService.*` | 通用媒体服务入口，封装 `MediaSession` 生命周期 |
| `MediaSession` | `src/video/MediaSession.*` | 音视频解复用、音频解码、视频分发、状态与时钟同步 |
| `VideoSession` | `src/video/VideoSession.*` | 视频包解码、帧缓存、渲染器绑定 |
| `VideoRendererGL` | `src/video/VideoRendererGL.*` | OpenGL 视频渲染 |

## 3. 当前实际运行路径
虽然仓库中保留了 `MediaService` 这条通用媒体入口，但当前视频 UI 的主运行路径仍然是：

1. `VideoListWidget` 通过 `VideoListViewModel` 拉取视频列表；
2. 用户点击某个视频后，请求视频流地址；
3. `VideoListWidget` 拿到流地址后打开 `VideoPlayerWindow`；
4. `VideoPlayerWindow::loadVideo()` 直接创建 `MediaSession`；
5. `MediaSession::loadSource()` 初始化 demux、音频流和 `VideoSession`；
6. `VideoSession` 绑定 `VideoRendererGL`；
7. 播放、暂停、seek、倍速等控制在 `VideoPlayerWindow` 中统一转调 `MediaSession`。

因此，当前视频链可以理解为“列表页 -> 播放窗口 -> 媒体会话”，而不是所有 UI 都直接依赖 `MediaService`。

## 4. 当前数据同步链
### 4.1 列表到窗口
- `VideoListViewModel` 负责服务端数据；
- `VideoListWidget` 负责把列表点击动作转成“请求流地址 + 打开窗口”；
- 视频列表本身不管理播放状态。

### 4.2 窗口到媒体会话
`VideoPlayerWindow` 当前承担了视频播放器宿主的职责：
- 维护进度条、播放按钮、播放速率、渲染质量；
- 监听 `MediaSession` 的 `metadataReady`、`durationChanged`、`stateChanged`、`positionChanged`；
- 根据会话状态刷新窗口 UI。

### 4.3 会话到渲染器
`MediaSession` 负责：
- 打开媒体源；
- 识别音频流与视频流；
- 为视频流创建 `VideoSession`；
- 启动 demux loop；
- 音频写入 `AudioPlayer`，视频包推给 `VideoSession`；
- 同步播放位置与音视频时钟。

而 `VideoSession` 则继续完成：
- 视频包解码；
- 帧缓冲管理；
- 与 `VideoRendererGL` 的绑定。

## 5. 当前版本的关键特点
1. 视频播放仍是“独立窗口”形态，没有完全并入音频主播放页；
2. `MediaSession` 是音频和视频共存时的调度核心；
3. 视频窗口的当前实现更偏面向用户交互，而 `MediaService` 更像可复用的服务层入口；
4. 当前文档描述时应以 `VideoPlayerWindow::loadVideo()` 为真实 UI 播放主链。

## 6. 与旧版文档相比的修正
- 不再把所有视频播放都描述成 `MediaService` 主导；
- 明确 `VideoPlayerWindow` 是当前 UI 运行主链；
- 明确音频输出与视频渲染共享 `MediaSession` 编排，而不是两套完全分离的播放器。
