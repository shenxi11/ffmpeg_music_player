## 3. 架构分层与调用链
播放一次音频的主链路如下：
1. UI 调用 `PlaybackViewModel::play(url)`。
2. `PlaybackViewModel` 转调 `AudioService::play(url)`。
3. `AudioService` 创建会话并切换当前会话。
4. `AudioSession::play()` 设定写入 owner，启动解码。
5. `AudioDecoder` 产出 PCM + 时间戳，发 `decodedData`。
6. `AudioSession::onDecodedData()` 做时间戳归一化与流控，再写入 `AudioPlayer`。
7. `AudioPlayer` 播放线程消费 PCM，更新位置并发 `positionChanged`。
8. `AudioService` 转发位置/时长/元数据，`PlaybackViewModel` 更新 UI。

```cpp
bool AudioService::play(const QUrl& url)
{
    int foundIndex = m_playlist.indexOf(url);
    if (foundIndex >= 0) {
        m_currentIndex = foundIndex;
    } else {
        m_playlist.append(url);
        m_currentIndex = m_playlist.size() - 1;
    }
    emit currentIndexChanged(m_currentIndex);

    QString sessionId = createSession();
    AudioSession* session = getSession(sessionId);
    if (!session) return false;

    if (session->loadSource(url)) {
        switchToSession(sessionId);
        session->play();
        return true;
    }
    destroySession(sessionId);
    return false;
}
```

```cpp
void AudioService::switchToSession(const QString& sessionId)
{
    if (!m_currentSessionId.isEmpty() && m_currentSessionId != sessionId) {
        if (auto oldSession = currentSession()) {
            oldSession->stop();
        }
        destroySession(m_currentSessionId);
        AudioPlayer::instance().resetBuffer();
    }

    m_currentSessionId = sessionId;
    if (auto newSession = currentSession()) {
        newSession->setVolume(m_globalVolume);
    }
    emit currentSessionChanged(sessionId);
}
```
作用：会话切换时同步清理旧会话与共享缓冲，避免串音与旧状态残留。

### 4.2 会话层 owner 归属与接管

```cpp
void AudioSession::play()
{
    if (!m_active) {
        m_active = true;
        emit sessionStarted();
    }
    m_manualPaused = false;

    const QString currentOwner = m_player->writeOwner();
    const bool takingOverFromOtherOwner = !currentOwner.isEmpty() && currentOwner != m_sessionId;
    if (takingOverFromOtherOwner && m_player->isPlaying() && !m_player->isPaused()) {
        m_internalPausePending = true;
        m_player->pause();
    }

    m_player->setWriteOwner(m_sessionId);
    if (takingOverFromOtherOwner) {
        m_player->stop();
        m_player->resetBuffer();
        m_player->setCurrentTimestamp(0);
    }
    m_decoder->startDecode();
    m_isBuffering = true;
}
```

```cpp
void AudioSession::resume()
{
    m_pendingResumeSignal = true;
    m_manualPaused = false;

    const QString currentOwner = m_player->writeOwner();
    const bool takingOverFromOtherOwner = currentOwner != m_sessionId;
    if (takingOverFromOtherOwner && m_player->isPlaying() && !m_player->isPaused()) {
        m_internalPausePending = true;
        m_player->pause();
    }

    m_player->setWriteOwner(m_sessionId);
    if (takingOverFromOtherOwner) {
        qint64 resumeTimestamp = (m_lastPausedPlaybackMs > 0 ? m_lastPausedPlaybackMs : m_lastDecodedTimestampMs);
        m_player->stop();
        m_player->resetBuffer();
        m_player->setCurrentTimestamp(resumeTimestamp);
        m_decoder->seekTo(resumeTimestamp);
        m_isBuffering = true;
    }
    m_decoder->startDecode();
}
```
作用：owner 交接时重建时间轴和缓冲，保证恢复播放不会沿用另一媒体（视频/旧会话）的输出状态。

### 4.3 写入归属防串流（核心）

```cpp
void AudioPlayer::setWriteOwner(const QString& ownerId)
{
    std::lock_guard<std::mutex> lock(m_ownerMutex);
    if (m_writeOwner != ownerId) {
        const QString previousOwner = m_writeOwner;
        const bool isHandoff = !previousOwner.isEmpty() && previousOwner != ownerId;
        if (isHandoff) {
            if (m_buffer) {
                m_buffer->clear();
            }
            std::lock_guard<std::mutex> tsLock(m_timestampMutex);
            while (!m_timestampQueue.empty()) {
                m_timestampQueue.pop();
            }
            m_currentTimestamp = 0;
            m_playbackRate = 1.0;
        }
        m_writeOwner = ownerId;
    }
}
```

代码位置：`AudioPlayer.cpp:241`
```cpp
void AudioPlayer::writeAudioData(const QByteArray& data, qint64 timestampMs, const QString& ownerId)
{
    if (!m_buffer || data.isEmpty() || ownerId.isEmpty()) return;
    {
        std::lock_guard<std::mutex> lock(m_ownerMutex);
        if (m_writeOwner.isEmpty() || ownerId != m_writeOwner) {
            return;
        }
    }
    m_buffer->write(data.data(), data.size());
    {
        std::lock_guard<std::mutex> lock(m_timestampMutex);
        m_timestampQueue.push({timestampMs, data.size()});
    }
    m_cv.notify_one();
    emit bufferStatusChanged(bufferFillLevel());
}
```
作用：只允许 owner 写入共享播放器，彻底阻断跨会话/跨媒体混写。

### 4.4 时间戳归一化与进度一致性

```cpp
void AudioSession::onDecodedData(const QByteArray& data, qint64 timestampMs)
{
    if (!m_active || m_manualPaused || !isWriteOwnerActive()) return;

    qint64 normalizedTimestampMs = timestampMs;
    if (m_decoderTimestampNeedsProbe) {
        const qint64 delta = m_decoderTimestampBaseMs - timestampMs;
        if (qAbs(delta) > 5000) {
            m_decoderTimestampHasOffset = true;
            m_decoderTimestampOffsetMs = delta;
        } else {
            m_decoderTimestampHasOffset = false;
            m_decoderTimestampOffsetMs = 0;
        }
        m_decoderTimestampNeedsProbe = false;
    }
    if (m_decoderTimestampHasOffset) {
        normalizedTimestampMs = timestampMs + m_decoderTimestampOffsetMs;
    }
    m_lastDecodedTimestampMs = normalizedTimestampMs;
    m_player->writeAudioData(data, normalizedTimestampMs, m_sessionId);
}
```

## 5. 当前数据同步策略总结
- 会话隔离：每次播放通过 `AudioSession` 隔离上下文，减少状态污染。
- 共享输出归属控制：`AudioPlayer` 通过 owner 机制只接受当前会话写入。
- 时间轴双重校准：会话层做解码时间修正，播放器层做时钟连续推进。
- 流控保护：缓冲高水位暂停解码、低水位进入 buffering，恢复后继续播放。
- UI 单向同步：服务层信号统一向 ViewModel 汇聚，避免 UI 直接操作底层线程对象。
- 音视频冲突治理：全局状态机只保留一个焦点播放器处于活动态。

## 6. 建议的排障关注点
- 若出现“进度条跳变”：优先看 `AudioSession.cpp:403` 的 offset 探测与 `AudioPlayer.cpp:463` 的时钟重置调用顺序。
- 若出现“串音/混音”：优先看 `AudioPlayer.cpp:241` 是否发生非 owner 写入，及 `AudioPlayer.cpp:161` 交接是否完整清空。
- 若出现“UI状态与真实播放不一致”：优先看 `viewmodels/PlaybackViewModel.cpp:105` 的 `togglePlayPause()` 分支是否走到 service 实时状态。
