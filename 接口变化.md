## 🎵 音乐服务接口

### 4. 获取音乐列表

**旧版本:**
```
GET /files
```

**新版本:**
```
GET /files
```

**变化:** ⚠️ **响应格式优化（向后兼容）**

**旧版响应格式:**
```json
{
  "千里之外/千里之外.mp3": "254.28 seconds",
  "发如雪/发如雪.mp3": "299.73 seconds"
}
```

**新版响应格式:**
```json
[
  {
    "path": "千里之外/千里之外.mp3",
    "duration": "254.28 seconds",
    "artist": "周杰伦&费玉清",
    "cover_art_url": "http://slcdut.xyz:8080/uploads/千里之外/千里之外_cover.png"
  },
  {
    "path": "发如雪/发如雪.mp3",
    "duration": "299.73 seconds",
    "artist": "周杰伦",
    "cover_art_url": "http://slcdut.xyz:8080/uploads/发如雪/发如雪_cover.jpg"
  }
]
```

**新增字段:**
- ✅ `artist` - 艺术家信息
- ✅ `cover_art_url` - 专辑封面URL（如果存在）

**兼容性:** ⚠️ **格式变化：从对象改为数组**
- 旧客户端需要适配
- 新格式更易扩展

**⚠️ 重要说明:**
- `cover_art_url` 中的封面路径可能包含空格或中文字符
- 实际访问时需要URL编码：如 `千里之外_cover.png` → `%E5%8D%83%E9%87%8C%E4%B9%8B%E5%A4%96_cover.png`
- 浏览器会自动编码，但程序调用时需手动编码（使用 `encodeURIComponent()` 或类似函数）
- 音频流接口同理

## ⚠️ 重要：URL编码问题

### 问题描述

所有包含**中文字符**、**空格**或**特殊字符**的URL路径都需要进行URL编码，包括：
- 音频文件路径（`/uploads/千里之外/千里之外.mp3`）
- 专辑封面路径（`/uploads/千里之外/千里之外_cover.png`）
- 歌词文件路径（`/uploads/千里之外/lrc`）

### 正确的处理方式

#### 1. JavaScript/TypeScript (前端)

```javascript
// 方法1：手动编码每个路径段
const folder = "千里之外"
const filename = "千里之外.mp3"
const url = `http://localhost:8080/uploads/${encodeURIComponent(folder)}/${encodeURIComponent(filename)}`
// 结果: http://localhost:8080/uploads/%E5%8D%83%E9%87%8C%E4%B9%8B%E5%A4%96/%E5%8D%83%E9%87%8C%E4%B9%8B%E5%A4%96.mp3

// 方法2：处理完整路径
const path = "千里之外/千里之外.mp3"
const encodedPath = path.split('/').map(encodeURIComponent).join('/')
const url = `http://localhost:8080/uploads/${encodedPath}`

// 方法3：HTML标签会自动编码（推荐）
<audio src="http://localhost:8080/uploads/千里之外/千里之外.mp3" controls />
<img src="http://localhost:8080/uploads/千里之外/千里之外_cover.png" />
```

#### 2. Python

```python
from urllib.parse import quote

# 编码路径
path = "千里之外/千里之外.mp3"
encoded_path = '/'.join(quote(part, safe='') for part in path.split('/'))
url = f"http://localhost:8080/uploads/{encoded_path}"

# 或使用 requests（自动处理）
import requests
response = requests.get(f"http://localhost:8080/uploads/千里之外/千里之外.mp3")
```

#### 3. Qt/C++ (QNetworkAccessManager)

```cpp
// Qt 会自动编码
QString path = "千里之外/千里之外.mp3";
QUrl url = QUrl::fromUserInput("http://localhost:8080/uploads/" + path);
// url.toString() 会自动编码为正确格式

// 或手动编码
QString encoded = QUrl::toPercentEncoding(path, "/");
```

#### 4. curl 命令行

```bash
# 方法1：使用引号（某些shell会自动处理）
curl "http://localhost:8080/uploads/千里之外/千里之外.mp3"

# 方法2：手动编码
curl "http://localhost:8080/uploads/%E5%8D%83%E9%87%8C%E4%B9%8B%E5%A4%96/%E5%8D%83%E9%87%8C%E4%B9%8B%E5%A4%96.mp3"
```

### 常见错误

❌ **错误1：直接拼接中文URL**
```javascript
const url = "http://localhost:8080/uploads/" + path  // path包含中文
fetch(url)  // 可能失败
```

❌ **错误2：过度编码**
```javascript
const url = encodeURIComponent("http://localhost:8080/uploads/千里之外/千里之外.mp3")
// 连 :// 和 / 都被编码了
```

✅ **正确：只编码路径部分**
```javascript
const base = "http://localhost:8080/uploads/"
const path = "千里之外/千里之外.mp3"
const encodedPath = path.split('/').map(encodeURIComponent).join('/')
const url = base + encodedPath
```

#### 1. `/files` 接口

**旧代码:**
```javascript
fetch('/files')
  .then(res => res.json())
  .then(data => {

      console.log(path, data[path])
    })
  })
```

**新代码:**
```javascript
fetch('/files')
  .then(res => res.json())
  .then(data => {
    // data 是数组: [{path, duration, artist, cover_art_url}, ...]

      console.log(item.path, item.duration, item.artist)
      if (item.cover_art_url) {
        console.log('封面:', item.cover_art_url)
      }
    })
  })
```
