# 播放历史删除接口文档

## 概述

新增了两个播放历史删除接口，用于管理用户的播放历史记录：
1. **批量删除接口**：删除指定的播放历史记录（支持批量）
2. **清空接口**：清空用户的全部播放历史

---

## 1. 批量删除播放历史

### 接口信息
- **URL**: `/user/history/delete`
- **方法**: `POST`
- **描述**: 删除用户指定的播放历史记录，支持批量删除

### 请求参数

#### Header 或 Query 参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_account | string | 是 | 用户账号（可通过Header的X-User-Account或Query参数传递） |

#### Body 参数（JSON）
```json
{
  "music_paths": [
    "D:/Music/告白气球/告白气球.mp3",
    "E:/MP3/爱情废柴/爱情废柴.mp3"
  ]
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| music_paths | array[string] | 是 | 要删除的音乐路径列表，支持批量删除 |

### 响应示例

#### 成功响应
```json
{
  "success": true,
  "message": "删除成功",
  "deleted_count": 15
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 是否成功 |
| message | string | 提示消息 |
| deleted_count | int | 实际删除的记录数 |

#### 错误响应
```json
缺少用户账号
```
```json
音乐路径列表不能为空
```

### 请求示例

#### cURL
```bash
# 通过Query参数传递用户账号
curl -X POST "http://slcdut.xyz:8080/user/history/delete?user_account=root" \
  -H "Content-Type: application/json" \
  -d '{
    "music_paths": [
      "D:/Music/告白气球/告白气球.mp3",
      "E:/MP3/爱情废柴/爱情废柴.mp3"
    ]
  }'

# 通过Header传递用户账号
curl -X POST "http://slcdut.xyz:8080/user/history/delete" \
  -H "X-User-Account: root" \
  -H "Content-Type: application/json" \
  -d '{
    "music_paths": [
      "http://slcdut.xyz:8080/uploads/给我一首歌的时间/给我一首歌的时间.mp3"
    ]
  }'
```

### 业务逻辑
1. 验证用户账号是否存在
2. 验证音乐路径列表是否为空
3. 批量删除指定路径的所有播放历史记录（可能有多条相同音乐的播放记录）
4. 返回实际删除的记录数

### 技术实现
- **Repository层**：使用 `DELETE ... WHERE user_account = ? AND music_path IN (?, ?, ...)` SQL语句批量删除
- **Service层**：参数验证和业务逻辑处理
- **Handler层**：HTTP请求处理和响应封装

## 2. 清空全部播放历史

### 接口信息
- **URL**: `/user/history/clear`
- **方法**: `POST`
- **描述**: 清空用户的全部播放历史记录

### 请求参数

#### Header 或 Query 参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_account | string | 是 | 用户账号（可通过Header的X-User-Account或Query参数传递） |

#### Body 参数
无需Body参数

### 响应示例

#### 成功响应
```json
{
  "success": true,
  "message": "清空成功",
  "deleted_count": 23
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| success | boolean | 是否成功 |
| message | string | 提示消息 |
| deleted_count | int | 实际删除的记录数 |

#### 错误响应
```json
缺少用户账号
```

### 请求示例

#### cURL
```bash
# 通过Query参数传递用户账号
curl -X POST "http://slcdut.xyz:8080/user/history/clear?user_account=root"

# 通过Header传递用户账号
curl -X POST "http://slcdut.xyz:8080/user/history/clear" \
  -H "X-User-Account: root"
```

### 业务逻辑
1. 验证用户账号是否存在
2. 删除该用户的所有播放历史记录
3. 返回实际删除的记录数

### 技术实现
- **Repository层**：使用 `DELETE FROM user_play_history WHERE user_account = ?` SQL语句删除全部记录
- **Service层**：参数验证和业务逻辑处理
- **Handler层**：HTTP请求处理和响应封装

## 使用场景

### 场景1：清理重复的播放记录
用户可能同一首歌播放了多次，想要删除某首歌的所有播放记录：
```bash
curl -X POST "http://slcdut.xyz:8080/user/history/delete?user_account=root" \
  -H "Content-Type: application/json" \
  -d '{
    "music_paths": ["D:/Music/告白气球/告白气球.mp3"]
  }'
```
**结果**：删除该歌曲的所有播放记录（可能有多条）

### 场景2：批量删除多首歌的播放记录
```bash
curl -X POST "http://slcdut.xyz:8080/user/history/delete?user_account=root" \
  -H "Content-Type: application/json" \
  -d '{
    "music_paths": [
      "D:/Music/告白气球/告白气球.mp3",
      "E:/MP3/爱情废柴/爱情废柴.mp3",
      "D:/Music/发如雪/发如雪.mp3"
    ]
  }'
```
**结果**：一次性删除多首歌的所有播放记录

### 场景3：清空全部播放历史
用户想要重置播放历史，清空所有记录：
```bash
curl -X POST "http://slcdut.xyz:8080/user/history/clear?user_account=root"
```
**结果**：删除该用户的所有播放历史记录

## 注意事项

1. **批量删除的影响范围**：
   - `DeletePlayHistory` 会删除指定音乐路径的**所有**播放记录
   - 例如：同一首歌播放了10次，调用一次删除接口会删除这10条记录

2. **删除计数**：
   - `deleted_count` 返回实际删除的数据库记录数
   - 如果某个路径在播放历史中不存在，不影响其他路径的删除

3. **不可恢复**：
   - 删除操作是永久性的，无法撤销
   - 建议在客户端实现二次确认

4. **权限控制**：
   - 当前通过 `user_account` 参数识别用户
   - 只能删除自己的播放历史

5. **清空操作的谨慎性**：
   - `ClearPlayHistory` 会删除用户的所有播放历史
   - 建议在UI上添加明显的警告和确认提示
