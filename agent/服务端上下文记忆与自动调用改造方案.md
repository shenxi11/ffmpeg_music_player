# 服务端上下文记忆与自动调用改造方案

## 1. 目标

把当前“能发 `tool_call` 的聊天后端”升级成“能记住上下文、能自动续调工具、能跨轮承接对象引用”的音乐 Agent。

本轮落地优先解决三个高频问题：

- 用户说“帮我查询我的流行歌单”时，服务端不再只回复口头说明，而是自动发起 `getPlaylists`
- 用户说“看看流行歌单里有什么歌”时，服务端能自动 `getPlaylists -> getPlaylistTracks`
- 用户说“看看这个歌单里有什么歌”“播放这个歌单”时，服务端能复用最近一次命中的歌单上下文

## 2. 当前问题

当前后端已经具备：

- `tool_call / tool_result`
- `clarification_request`
- `plan_preview / approval_request / progress / final_result`
- SQLite 消息存储与事件审计

但自动执行仍不稳定，核心原因有两点：

1. 意图识别仍以少量硬编码规则为主，覆盖面不够
2. 工作记忆只保存了候选列表和 pending tool，缺少“当前目标”和“最近命中的对象”

## 3. 改造原则

- Agent 不直接操作 Qt 控件，只通过 Host API 工具协议驱动客户端
- 查询类和播放类默认自动执行，不先回礼貌话术
- 多候选场景进入 `clarification_request`
- 写操作继续走 `plan + approval`

## 4. 分阶段计划

### 第一阶段：补齐查询链路与基础上下文

目标：

- 支持 `query_playlist`
- 支持 `inspect_playlist_tracks`
- 支持 `get_recent_tracks`
- 记住最近命中的歌单和歌曲

改造点：

- 扩展 `WorkflowState`
- 扩展 `ToolRegistry`
- 扩展 `MusicAgentRuntime`
- 补测试覆盖自然说法和跨轮引用

### 第二阶段：抽出“工具结果后继续决策”的通用循环

目标：

- 降低 `handle_tool_result()` 中大量 if/else 分支
- 把 “integrate observation” 和 “decide next action” 分离

### 第三阶段：引入持久化上下文快照

目标：

- 服务重启或 Qt 重连后恢复最近一次上下文
- 为“这个歌单”“这首歌”“刚才那个”提供更稳定的跨轮记忆

## 5. 第一阶段具体设计

### 5.1 新增/强化的工作记忆

- `current_goal`
- `goal_status`
- `resolved_entities`
- `recent_playlist_tracks`
- `recent_recent_tracks`
- `last_named_playlist`
- `last_named_track`
- `last_tool_observation`

### 5.2 第一阶段新增自动能力

#### 查询指定歌单

示例：

- 帮我查询我的流行歌单
- 看看有没有流行歌单
- 查一下流行歌单

执行链：

- `getPlaylists`
- 服务端本地匹配
- 单命中则直接总结
- 多命中则 `clarification_request`

#### 查看歌单内容

示例：

- 看看流行歌单里有什么歌
- 查看流行歌单的歌曲
- 这个歌单里有什么歌

执行链：

- 若消息里带歌单名：`getPlaylists -> getPlaylistTracks`
- 若引用最近命中的歌单：直接 `getPlaylistTracks`

#### 查看最近播放

示例：

- 最近听了什么
- 最近播放了什么
- 看看我的播放历史

执行链：

- `getRecentTracks`
- 服务端汇总并写入上下文

## 6. 与 Qt 端联调的影响

第一阶段不需要新增 WebSocket 协议类型，继续复用：

- `tool_call`
- `tool_result`
- `clarification_request`
- `assistant_start / assistant_chunk / assistant_final`

需要 Qt 端确保这两个工具返回结构稳定：

- `getPlaylistTracks`
- `getRecentTracks`

建议返回结构：

```json
{
  "playlist": {
    "playlistId": "playlist-1",
    "name": "流行歌单",
    "trackCount": 18
  },
  "items": [
    {
      "trackId": "track-1",
      "title": "晴天",
      "artist": "周杰伦"
    }
  ]
}
```

```json
{
  "items": [
    {
      "trackId": "track-1",
      "title": "晴天",
      "artist": "周杰伦"
    }
  ]
}
```

## 7. 验收标准

- “帮我查询我的流行歌单” 能自动触发 `getPlaylists`
- “看看流行歌单里有什么歌” 能自动触发 `getPlaylists -> getPlaylistTracks`
- “看看这个歌单里有什么歌” 能复用最近命中的歌单
- “最近听了什么” 能触发 `getRecentTracks`
- 多候选歌单会进入澄清，而不是随便选一个
- 原有播放歌曲、播放歌单、创建歌单链路不回退
